# cratestack-policy

Policy predicate types and evaluation for CrateStack access control.

## Overview

`cratestack-policy` defines the predicate language used in `@@allow` and `@@deny` model attributes. It provides compile-time generated predicates that evaluate against `CoolContext` to determine read/write permissions.

## Installation

```toml
[dependencies]
cratestack-policy = "0.2"
```

## Usage

Predicates are typically generated by `include_schema!` from schema attributes:

```cstack
model Post {
  id String @id
  title String
  authorId String
  published Boolean
  
  @@allow("read", auth() != null)
  @@allow("update", auth().id == authorId)
  @@allow("delete", auth().role == "admin")
}
```

## Predicate Types

### Authentication

```rust
use cratestack_policy::ReadPredicate;

ReadPredicate::AuthNotNull;  // Requires authenticated user
ReadPredicate::AuthIsNull;    // Requires anonymous access
```

### Roles and Tenants

```rust
ReadPredicate::HasRole { role: "admin" };
ReadPredicate::InTenant { tenant_id: "org_123" };
```

### Field Comparisons

```rust
use cratestack_policy::{ReadPredicate, PolicyLiteral};

// Compare field to auth claim
ReadPredicate::FieldEqAuth {
    column: "authorId",
    auth_field: "id",
};

// Compare field to literal
ReadPredicate::FieldEqLiteral {
    column: "status",
    value: PolicyLiteral::String("active"),
};

// Boolean field check
ReadPredicate::FieldIsTrue { column: "published" };
```

### Compound Expressions

```rust
use cratestack_policy::{ReadPolicy, PolicyExpr};

let policy = ReadPolicy {
    expr: PolicyExpr::And(&[
        PolicyExpr::Predicate(ReadPredicate::AuthNotNull),
        PolicyExpr::Or(&[
            PolicyExpr::Predicate(ReadPredicate::HasRole { role: "admin" }),
            PolicyExpr::Predicate(ReadPredicate::FieldEqAuth {
                column: "ownerId",
                auth_field: "id",
            }),
        ]),
    ]),
};
```

### Relation-Based Predicates

```rust
use cratestack_policy::{ReadPredicate, RelationQuantifier};

// Check all related records satisfy predicate
ReadPredicate::Relation {
    quantifier: RelationQuantifier::Every,
    parent_table: "Account",
    parent_column: "id",
    related_table: "Transfer",
    related_column: "sourceId",
    expr: &PolicyExpr::Predicate(ReadPredicate::FieldEqAuth {
        column: "destinationId",
        auth_field: "accountId",
    }),
};
```

## Quantifiers

| Quantifier | Meaning |
|------------|---------|
| `ToOne` | Relation must point to exactly one record |
| `Some` | At least one related record satisfies predicate |
| `Every` | All related records satisfy predicate |
| `None` | No related records satisfy predicate |

## Procedure Policies

Procedures can have policies too:

```cstack
mutation procedure transfer(amount: Decimal, to: String): Transfer
  @isolation("serializable")
{
  @@allow("execute", auth().role == "admin")
}
```
