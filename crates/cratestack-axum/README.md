# cratestack-axum

Axum integration for generated routes and HTTP transport.

## Overview

`cratestack-axum` provides the transport glue that generated Axum routers depend on: codec/transport negotiation helpers, query-string parsing for filter expressions, idempotency and rate-limit Tower layers, and version (ETag / If-Match) helpers for optimistic locking.

The crate re-exports `axum` so a service can pull a single dependency.

## Installation

```toml
[dependencies]
cratestack-axum = "0.2.2"
```

## Generated Router

Generated by `include_server_schema!`. The bound router takes the runtime, a codec implementing `CoolCodec`, and an `AuthProvider`:

```rust
use cratestack::include_server_schema;
use cratestack_codec_cbor::CborCodec;

include_server_schema!("schema.cstack");

let pool = sqlx::PgPool::connect(&database_url).await?;
let cool = cratestack_schema::Cratestack::builder(pool).build();

let router = cratestack_schema::axum::model_router(
    cool,
    CborCodec,
    AppAuthProvider, // implements cratestack::AuthProvider
);

let app = axum::Router::new().nest("/api", router);
```

## Transport Helpers

The crate exposes header validation, codec encode/decode, and transport-level encode/decode helpers built around the `HttpTransport` trait. Public entry points include:

- `validate_codec_request_headers` / `validate_codec_response_headers`
- `validate_transport_request_headers` / `validate_transport_response_headers`
- `decode_codec_request`, `encode_codec_response`, `encode_codec_result`
- `decode_transport_request_for`, `encode_transport_result`, `encode_transport_sequence_result`
- `parse_if_match_version`, `set_version_etag`
- `parse_traceparent`, `parse_client_ip`, `enrich_context_from_headers`

The `CBOR_SEQUENCE_CONTENT_TYPE` constant is exported as `application/cbor-seq`; sequence-mode transport is the documented target but is not enabled on generated routers today (a single configured codec per router is the current behaviour).

## Idempotency

Use `IdempotencyLayer` together with any `IdempotencyStore`. `SqlxIdempotencyStore` (from `cratestack-sqlx`) and `RedisIdempotencyStore` (from `cratestack-redis`) are the bundled implementations.

```rust
use std::sync::Arc;
use std::time::Duration;
use cratestack_axum::idempotency::IdempotencyLayer;
use cratestack_sqlx::SqlxIdempotencyStore;

let store = Arc::new(SqlxIdempotencyStore::new(pool.clone()));
let app = axum::Router::new()
    .nest("/api", router)
    .layer(IdempotencyLayer::new(store, Duration::from_secs(24 * 60 * 60)));
```

`with_principal_fingerprint` overrides the default authorization-header fingerprint used to scope keys per principal.

The crate exports `IDEMPOTENCY_TABLE_DDL` for the SQL store; see the [Idempotency guide](https://cratestack.dev/guides/idempotency) for the table contract and replay semantics.

## Rate Limiting

Token-bucket layer with a pluggable store. `InMemoryRateLimitStore` is the default; production multi-replica clusters back this with Redis.

```rust
use std::sync::Arc;
use cratestack_axum::ratelimit::{InMemoryRateLimitStore, RateLimitConfig, RateLimitLayer};

let store = Arc::new(InMemoryRateLimitStore::new());
let app = axum::Router::new()
    .nest("/api", router)
    .layer(RateLimitLayer::new(store, RateLimitConfig::new(100, 10.0)));
```

`RateLimitConfig` carries `burst` (max in-flight) and `refill_per_second`. `RateLimitDecision` is either `Allowed { remaining }` or `Throttled { retry_after_secs }`.

## Query Parsing

`parse_filter_expression` parses a single filter expression into a `QueryExpr`; `parse_query_pairs` returns the raw query-string pairs already URL-decoded.

```rust
use cratestack_axum::{parse_filter_expression, QueryExpr};

let expr: QueryExpr = parse_filter_expression("published = true AND authorId = 'usr_1'")?;
```

## Content Types

| Content Type            | Codec / framing             |
|-------------------------|-----------------------------|
| `application/cbor`      | CBOR single item            |
| `application/json`      | JSON single item            |
| `application/cbor-seq`  | CBOR sequence (target, not yet enabled on generated routes) |
