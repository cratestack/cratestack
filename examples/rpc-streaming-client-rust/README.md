# rpc-streaming-client-rust-example

Rust binary that consumes an `application/cbor-seq` stream from a `transport rpc` server via a typed client generated by `include_client_schema!`. Companion to [`rpc-streaming-example`](../rpc-streaming/), which serves the other side of the wire from the same `.cstack` schema.

## What it shows

- **Typed streaming method generation.** The schema declares `procedure ticks(args: TickerArgs): Tick[]` ‚Äî return arity `T[]` makes the macro emit a streaming method on the generated client. `client.procedures().ticks(&args)` returns `RpcStream<Tick>` (= `tokio::sync::mpsc::Receiver<Result<Tick, RpcClientError>>`). No hand-defined wire types in this example.
- **Streaming-from-first-byte.** Items arrive on the channel as cbor-seq frames parse off the wire ‚Äî no full-body buffering. Time-to-first-item is the time to one chunk.
- **The error split.** Non-2xx responses (auth failure, unknown op, server error) surface from the outer `Result` before the channel ever opens. Per-item failures (mid-stream decode error, transport reset) appear as terminal `Err` items on the channel.
- **Auth via `RequestAuthorizer`.** `CratestackClient::with_request_authorizer(...)` on the runtime flows through to every generated method automatically ‚Äî no per-call header argument needed.

## Run

Two terminals:

```bash
# Terminal 1 ‚Äî start the server example:
cargo run -p rpc-streaming-example

# Terminal 2 ‚Äî consume the stream:
REMOTE_URL=http://localhost:3001 cargo run -p rpc-streaming-client-rust-example
```

Expected output (Terminal 2):

```
Streaming `procedure.ticks` from http://localhost:3001/ (start=100, count=10):

  index=0    value=100
  index=1    value=101
  ‚Ä¶
  index=9    value=109

stream closed cleanly after 10 items
```

## Tests

`tests/smoke.rs` spawns a tiny in-process axum server that emits cbor-seq frames one-at-a-time and exercises:

- `streams_each_tick_as_it_arrives` ‚Äî all items arrive in order; channel closes cleanly.
- `missing_auth_header_surfaces_as_remote_error_before_stream_opens` ‚Äî 401 from the server surfaces as `Err` from the generated method before any channel is opened.

Run with `cargo test -p rpc-streaming-client-rust-example`. Self-contained ‚Äî does not require the server example to be running.
