# embedded-cli

Desktop note-taking CLI on top of [`cratestack-rusqlite`](../../crates/cratestack-rusqlite/). Demonstrates the file-backed-SQLite use case (laptops, servers, internal tools).

## What it shows

- `include_embedded_schema!("schema.cstack")` for compile-time model + descriptor + delegate code-gen
- `RusqliteRuntime::open(path)` against a file on disk
- `ModelDelegate::new(...).create(...)` / `.find_many()` / `.update(...)` / `.delete(...)` end-to-end
- Filter helpers (`note::pinned().is_true()`, `note::completed().is_false()`) generated by the macro
- Order-by + limit on a `find_many` query

The same `include_embedded_schema!`-generated code compiles unchanged for mobile FFI bridges and `wasm32-unknown-unknown` browser targets — only the runtime open path differs.

## Run

```bash
cargo run -p embedded-cli-example -- --db /tmp/notes.db add "Buy milk" --pinned
cargo run -p embedded-cli-example -- --db /tmp/notes.db add "Write docs" --body "Cover wasm + Vite"
cargo run -p embedded-cli-example -- --db /tmp/notes.db list
cargo run -p embedded-cli-example -- --db /tmp/notes.db list --pinned
cargo run -p embedded-cli-example -- --db /tmp/notes.db list --open
cargo run -p embedded-cli-example -- --db /tmp/notes.db done <uuid>
cargo run -p embedded-cli-example -- --db /tmp/notes.db delete <uuid>
cargo run -p embedded-cli-example -- --db /tmp/notes.db count
```

Or build the binary once and use the shorter form:

```bash
cargo build -p embedded-cli-example
./target/debug/notes --db /tmp/notes.db add "First note"
```

## Tests

The crate ships a `tests/smoke.rs` that exercises the same delegate calls against an in-memory database. Runs in CI via the standard `cargo test --workspace`:

```bash
cargo test -p embedded-cli-example
```

## Schema

See [`schema.cstack`](schema.cstack). A single `Note` model with id, title, body, two boolean flags, and timestamps — deliberately minimal so the focus stays on the delegate API rather than schema features.

## Notes

- `cratestack` is currently a transitive dep because `include_embedded_schema!` still emits `::cratestack::*` paths for shared SQL helpers (`IntoSqlValue`, `FilterExpr`, ...). A 0.3.x follow-up will switch the macro to emit `::cratestack_core::*` + `::cratestack_sql::*` directly, removing the facade requirement and shedding the unused sqlx/axum weight for purely embedded consumers.
